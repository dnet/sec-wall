    URL rewriting

    How it works

    sec-wall has support for changing the URLs on fly, that is, the URL a client application is accessing doesn't necessarily have to map 1:1 to what a backend server exposes, there's always an option of sec-wall's adjusting it to your needs. You doubtless remember that parts of the URLs to be matched need to be named and the names can be used for constructing new URLs, like in this example:
    That is, it's possible to state that a pattern of the URL the client application invokes - such as /before/\/profile/\/ gets mapped into a backend server's /after/{customer_id}/cust-profile/{customer_name}/ pattern. Hence in runtime, a client application's invoking the /before/2816917/profile/Brian Ó Nualláin/ URL will result in sec-wall's invoking the backend's URL of /after/2816917/cust-profile/Brian Ó Nualláin/
